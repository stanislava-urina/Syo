Telegram Bot для записи в очередь (Queue Management System)

Telegram бот для интеграции с системой управления очередью (QMS). Позволяет пользователям записываться в живую очередь или на конкретное время, просматривать историю записей и получать напоминания.

Функциональность

Для пользователей
- Регистрация через отправку контакта (номер телефона)
- Запись в живую очередь - получение талона с примерным временем ожидания
- Запись на конкретное время - выбор доступных слотов
- Выбор категорий услуг (при необходимости)
- История записей - просмотр активных и завершенных записей
- Отмена записи - возможность отменить активную запись
- Напоминания - уведомление за 15 минут до записи
- Ограничение на активные записи - не более 2 активных записей на пользователя

Для администраторов
- Панель администратора (доступ по команде /admin)
- Управление бан-листом - блокировка пользователей по ID
- Редактирование информации - тексты "О компании" и "Контакты"
- Уведомления о новых записях - моментальное оповещение всех админов
- Уведомления об отменах - информация о отмененных записях

Установка и запуск

Требования
- Python 3.8+
- Установленные зависимости

Установка зависимостей
pip install aiogram requests

Файлы конфигурации
Бот использует текстовые файлы для хранения данных:
- admin.txt - список ID администраторов (по одному на строку)
- banned.txt - список забаненных пользователей
- registrated.txt - словарь зарегистрированных пользователей (ID:телефон)
- Hi.txt - приветственное сообщение для команды /start
- Abot.txt - информация о компании
- contacts.txt - контактная информация
- history_{user_id}.json - история записей пользователя

Настройка токенов
В коде необходимо заменить:
BOT_TOKEN = "ваш_токен_бота"
API_TOKEN = "ваш_токен_api_qms"

Запуск
python bot.py

Структура проекта

bot.py              # Основной файл бота

Команды бота

Пользовательские
- /start - начало работы, приветствие
- /help - показать меню

Административные
- /admin - вход в панель администратора

Основные функции

Регистрация
Пользователь отправляет свой контакт, бот сохраняет номер телефона и предоставляет доступ к функционалу.

Запись на время
1. Выбор услуги из списка
2. Выбор категорий (опционально)
3. Выбор доступного временного слота
4. Ввод номера автомобиля
5. Подтверждение записи
6. Получение номера талона

Живая очередь
1. Выбор услуги
2. Автоматическая регистрация в очереди
3. Получение номера талона и примерного времени ожидания

История записей
- Отображение активных и завершенных записей
- Возможность отмены активных записей
- Автоматическое обновление статусов

Лимиты и ограничения
- Максимум 2 активные записи на одного пользователя
- Живая очередь и запись на время считаются в общем лимите
- Напоминания отправляются только для записей на время

Логирование
Бот ведет подробное логирование в файл bot.log с ротацией:
- Максимальный размер файла: 5 МБ
- Количество хранимых файлов: 5
- Уровень логирования: INFO

Обработка ошибок
- Автоматическое восстановление при сбоях API
- Тестовые ответы при недоступности сервера
- Защита от поврежденных файлов истории

Безопасность
- Проверка прав доступа для административных функций
- Бан-лист для блокировки нежелательных пользователей
- Валидация вводимых данных (номер автомобиля)
- Защита от превышения лимита записей
